Wo ist das Problem?

Jede IDE hat eigene Style-Vorgaben.

Zeichensatz und newlines können mit Betriebssystem variieren.

Beispiele

Newlines 
        
            // Linux style newline
            print("Hello World")\n

            // Windows style newline
            print("Hello World")\r\n
Erkläre:
LF - Linefeed \n
CR - Carriage return \r
GIT kann unter Windows automatisch konvertieren

Datei mit Newlines unter Linux

        
            var name = "Max Mustermann";\n
            var firstname = "";\n
            var idx = 0;\n
            while (name[idx] != " ") {\n
              firstname += name[idx];\n
              idx = idx + 1;\n
            }\n
Ein Teammitglied benutzt Windows, ein anderes Linux

Darstellung unter Linux

        
            var name = "Max Mustermann";
            var firstname = "";
            var idx = 0;
            while (name[idx] != " ") {
              firstname += name[idx];
              idx = idx + 1;
            }

Darstellung unter Windows

        
            var name = "Max Mustermann";var firstname = "";var idx
            = 0;firstname += name[idx];idx = idx + 1;}

Lösung: EditorConfig

https://editorconfig.org

Lösung: EditorConfig

        
          # Unix-style newlines with a newline ending every file
          [*]
          end_of_line = lf
          insert_final_newline = true
          charset = utf-8

          # 4 space indentation
          [*.py]
          indent_style = space
          indent_size = 4
